# Yoga AI Coach – Product Notes

端到端瑜珈姿勢辨識 PoC × 互動回饋

---

## S ｜人數縮水，題目半空

- 專案動機：在家運動者常缺乏即時回饋與調整，希望用**低成本方式**協助練習者檢查姿勢是否落在合理範圍。
- 初始 7 人團隊對題目方向分歧；我在組員流動、題目架構未全時加入，必須釐清「人力與技術到底能做到哪裡」。

---

## T & A ｜題目收斂與技術落地

### 1. 題目收斂：把模糊的「AI 瑜珈教練」變成可驗證的 PoC

- 在站立會議主動發言，和同學把需求收斂成：
  - **裝置**：只用一般攝影機，不依賴穿戴裝置或專用硬體。
  - **回饋目標**：先做到「姿勢分類 + 角度判斷」的基礎回饋。
  - **範圍**：以少數幾個常見姿勢做技術可行性驗證（PoC）。

### 2. 資料流程協作：把 pipeline 說清楚

- 與 ETL 同學對齊並文件化資料規格：

  `影片 → MediaPipe 33 點 → 27 維幾何特徵 → 標準化`

- 涵蓋 300+ 段影片、每段固定 30 幀，明確紀錄取樣方式與限制，方便之後重現與調整。

### 3. 模型實驗（DL Model Owner）

- 善用 ChatGPT / Claude / DeepSeek，產出 LSTM-Attention 與 Encoder-only Transformer baseline 程式。
- 把原本散在 Colab 的程式整理成可重複使用的 `.py` 腳本，補上註解與參數說明。
- 手動嘗試多組超參數，在小資料與本地算力受限的情況下，訓練出：
  - **5 類姿勢 Top-1 準確率約 95.7 % 的基準模型**。

### 4. 使用情境測試：實際摸清使用邊界

- 設計單人 / 雙人 / 多人、不同距離與背景的測試情境，實際用 UI 操作。
- 根據測試結果回推目前可接受的 **MVP 使用邊界**：
  - 限定 **單人**、**背景不複雜** 的情境。
  - 將原本 6 類動作調整為**可穩定辨識的 5 類姿勢**。

### 5. 練習紀錄欄位設計：為未來分析留鉤子

- 與 UI 同學一起討論練習紀錄需要哪些欄位。
- 決定將「**姿勢名稱、維持秒數、時間戳記**」寫入 MySQL，
  為後續的運動歷程分析與個人化建議預留基礎資料。

## R ｜競品觀察與產品展望

### 1. 市場觀察：我們卡在誰沒做到的空隙？

- **穿戴裝置**：多數健身手環與裝置很少專門針對瑜珈；
  姿勢複雜、需求較小眾，讓這塊市場處於真空狀態。
- **健身鏡**：具即時偵測與指導能力，但硬體成本與安裝門檻高，
  比較偏向高價展示與特定場域，而非一般人在家就能使用。
- **智慧瑜珈墊／瑜珈褲**：多數只能感測接觸點壓力，對「全身姿勢」的理解有限。

> 本專案驗證的是：在**低成本裝置**與**有限算力**下，  
> 仍可在本地端運行、提供角度輔助的姿勢辨識 PoC。

### 2. 目前可提供的價值（現階段）

- 在 **筆電 / 平板本地端即可運作**，不需連網或額外硬體。
- 目前支援 **5 類瑜珈姿勢辨識**，搭配角度輔助指標與簡單練習紀錄。
- 比較適合**已有基礎瑜珈經驗**的練習者，用來快速檢查當下動作是否落在合理範圍。

### 3. 可能的產品擴展方向（未來想像）

- **知識體系與姿勢庫**
  - 增加更多姿勢類別與標準示範資料。
  - 結合每個姿勢對應的身心狀態小提示，讓回饋不只停在「對或錯」。
- **練習日誌與數據分析**
  - 把現有 MySQL 紀錄延伸成完整的練習日誌與圖表，
    支援規律訓練規劃、留存分析與產品迭代。
- **社群與場館串接**
  - 初階：產生練習紀錄卡片與發文模板，讓使用者能一鍵打卡分享。
  - 進階：與瑜珈教室或品牌會員系統串接，用練習數據連結課程、活動與社群營運。
